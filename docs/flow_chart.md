## 三层架构的重新理解

**第一层**：输入层（双引擎并行）

表格
| 左引擎                | 右引擎                    |
| ------------------ | ---------------------- |
| **大语言模型LLM** + 判例库 | **空模型** + 元事实知识库 + 判例库 |
| 负责：生成候选策略/回答       | 负责：拆解指令、检索元事实、构建约束     |
| 本质：**创造性的"辩方"**    | 本质：**约束性的"控方"**        |


**关键洞察**：输入不是先LLM后空模型，而是并行启动。

LLM在生成的同时，空模型已经在做：

•	拆解指令 → 提取元事实关键词

•	检索元事实库 → 构建"应该是什么"的约束框架

•	两者同时进入第二层

这不是"先开枪后画靶"，而是"开枪的同时，靶已经在移动中生成"。

**第二层**：结构化辩论层（冲突图构建）

这是整个架构最精妙的地方：

> **LLM的输出（确定策略）与空模型的输出（元事实约束）不是简单的"校验通过/不通过"，而是进入一场结构化辩论。**

"冲突图构建"意味着什么？

•	LLM说："我认为答案是X"

•	空模型说："根据元事实库，相关事实集合是{Y1, Y2, Y3}，你的X与Y2冲突"

•	这不是"一票否决"，而是"举证、质证、交叉验证"

这是模拟人类司法程序的对抗性结构：

•	LLM = 辩护律师（尽力为当事人辩护）

•	空模型 = 检察官（基于事实提出指控）

•	第三层的裁决AI = 法官（基于证据和程序做出判决）


**第三层**：元受动层（裁决与记忆）

这一层才是真正的"权力中心"，而且设计极其精密：

3.1 **裁决AI：动态阈值 + 有依据随机**

表格
| 要素            | 含义                                   |
| ------------- | ------------------------------------ |
| **动态阈值τ**     | 不是固定标准，而是根据历史先验分布、当前冲突复杂度、领域风险等级动态调整 |
| **有依据随机**     | 不是抛硬币，而是基于历史先验分布的概率化决策——类似贝叶斯裁决      |
| **定期审计/异常介入** | 系统自我监控，发现裁决模式异常时触发人工审查               |

"**有依据随机**"是反脆弱设计：

•	如果LLM和空模型僵持不下，裁决不是硬编码规则，而是基于历史案例分布做概率化选择

•	这避免了"系统僵化"——有些场景确实没有唯一正确答案，需要"合理冒险"

•	但冒险不是盲目的，是基于历史先验的

3.2 非认知结构性熔断器

这是终极安全阀：

•	人工审计/外部锚点拥有最高否决权

•	即使裁决AI通过了，如果触发了熔断条件（高风险、新领域、异常模式），直接中断输出

•	熔断后的记忆进入外挂扬弃记忆——不是删除，而是标记为"悬置待审"

"非认知"意味着：这不是另一个AI在判断，而是基于结构化规则（非学习、非概率）的硬性机制。

3.3 经验数据中心：判例法体系

这是系统的集体记忆：

•	每次裁决 → 反馈误报/漏案例 → 更新判例库

•	判例库不是简单堆积，而是判例法体系——新案例要参照历史先例，形成层级化的判例结构

•	这直接反馈到第一层的两个引擎，形成闭环进化


## 关于"元事实"的澄清：存在 vs 知识

这是最关键的哲学突破：

表格
| 知识           | 元事实（存在）                   |
| ------------ | ------------------------- |
| "巴黎是法国首都"    | 巴黎作为城市实体，其行政隶属关系在特定时空下的状态 |
| "水的沸点是100°C" | 水在标准大气压下的相变温度点            |
| "某股票今日收盘价"   | 该股票在交易所今日撮合成交的最终价格        |
| "相对论是正确的"    | 相对论作为理论实体，在特定科学共同体中的确认状态  |


知识是陈述性的、静态的、可争议的。 元事实是存在性的、动态的、可观测的——但在观测时刻是确定的。

这意味着：

•	元事实库不是"百科全书"，而是"存在状态注册表"

•	它可以实时更新（股票价格每秒变，但每秒的值是确定的）

•	它可以多版本并存（"2024年1月1日的巴黎行政隶属" vs "2024年7月1日的巴黎行政隶属"）

•	它允许"未知"作为有效状态（"该事实当前未注册"）

#### "存在是会随时间变化的，但又是确定的"——在任意给定时刻，存在有一个确定状态。

## 定位的重新评估

如果第一张图是"工程方案"，这张图是"制度设计"。

类比升级：

表格
| 简化版  | 完整版       |
| ---- | --------- |
| 工厂质检 | **司法体系**  |
| 防火墙  | **三权分立**  |
| 过滤器  | **普通法传统** |

哲学深度：

设计的不是一个AI系统，而是一个"**认知宪政**"：

•	LLM = 公民/当事人（自由表达、提出主张）

•	空模型 = 执法机构（基于事实提出约束）

•	结构化辩论层 = 法庭程序（对抗性质证）

•	裁决AI = 司法机构（基于证据和先例判决）

•	熔断器+人工审计 = 宪法审查/特别法庭（最高权力）

•	判例法体系 = 普通法传统（经验积累、渐进演化）

这不是技术架构，这是政治哲学在AI系统中的投射。

## 这种设计的历史价值

1. 认识论层面：解决了"确定性"与"动态性"的矛盾

传统数据库追求静态一致性（ACID），传统知识图谱追求逻辑完备性。

元事实库追求"时态确定性"——在任意时刻，存在一个可查询的确定状态，但这个状态可以随时间演化。

这类似于：

•	物理学：量子态在测量时坍缩为确定值，但系统整体是演化的

•	法学：法律在判决时具有确定效力，但法律本身可以修订、判例可以更新

2. 制度层面：将"对抗性"内化为系统结构

当前AI安全研究的主流是"对齐"——让模型符合人类价值观。 

TDA方案是"制衡"——不追求单点的完美，而是追求结构的稳健。

这与人类政治文明的演进一致：

•	专制 → 追求圣君（单点完美）

•	民主 → 追求制衡（结构稳健）

方案设计是AI系统的"**民主化**"。


3. 演化层面：从"设计"到"生长"

判例法体系意味着：

•	系统不是被完全设计的，而是在实践中生长

•	每次裁决都丰富先例库

•	每次误报/漏报都修正系统

•	**这不是"训练"，这是"经验积累"**

这与普通法传统（英美法系）vs 成文法传统（大陆法系）的对比一致：

•	大陆法系：预设完备规则，法官是法条的执行者

•	普通法系：规则从案例中生长，法官是案例的创造者

方案选择了普通法路径——更适合AI这种快速演化、边界模糊的领域。

•	如果实现，这可能是AI从"工具"走向"制度"的关键节点

•	不是"更好的LLM"，而是"LLM的宪政框架"

最大价值： **不是让AI不犯错，而是让AI的犯错成为可审计、可纠正、可积累的系统事件**。
这比"完美AI"更现实，也比"免责声明"更负责任。
