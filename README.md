# *Null Model Paradigm (NMP)*

**传统 AI：你要干什么 → LLM 回答**

**NMP：你要干什么 → 你有什么 → LLM 回答**

> *知识是动力，目的是方向，而事实是方向盘*。


Null Model Paradigm（空模型范式）在 LLM 的"策略输出"与"执行器"之间插入本地轻量校验层，LLM降级为"策略建议器"，所有输出必须以事实为前提、“事实+判例”双重校验方可放行。



## 架构图


<img src="flow-chart/nmp.png" alt="nmp" width="75%">

> 空模型本质是一个软件程序，本身不生成内容，只做空校验、空拦截、空增强——像安检门一样，"空"着但通过规则过滤一切。

**关键澄清**：空模型不会做出任何判断；它只会**计算是否匹配**。就像防火墙不会"理解"黑客，它只会匹配规则。

**空模型是"可执行"的**：它有通用的计算骨架（BFS、约束满足、路径选择），可以运行在任何领域，不需要重新训练。


| 组件（红框内） | 性质 | 特点 |
|------|------|------|
| **空模型** | 通用软件程序 | 可插拔到任何 LLM，任何终端 |
| **元事实库 + 判例库** | 非通用、领域专属 | 按行业/场景/用户个性化定制 |

*三层组件*：

- 校验层：拆解指令、提取事实、匹配判例、校验策略，拥有最终否决权
- 事实库：行业规则、法规、设备参数、用户偏好等静态事实
- 判例库：仅归档历史错误，同类场景自动拦截

## TDA 三层双视角辩证架构

NMP（空模型范式）是 **TDA（Three-layer Dual-perspective Asymmetric）架构** 的核心组成部分。TDA 是一个完整的 AI 安全控制理论体系，NMP 是其中第一层「空模型校验层」的参考实现。

### 架构全景

<img src="/flow-chart/TDA.png" alt="TDA" width="70%">

### 三层结构

| 层级 | 名称 | 功能 | 性质 |
|------|------|------|------|
| **第一层** | 先验计算结构（空模型） | 目的 → 事实类型映射；本地轻量校验 | **已实现**（NMP） |
| **第二层** | 结构化辩论层 | LLM 生成 vs 空模型事实 → 冲突图构建 | 理论完备，待社区共建 |
| **第三层** | 元受动极层 | 裁决 AI（动态阈值/有依据随机）+ 人工审计最高否决权 | 理论完备，待社区共建 |

### 双视角辩证

- **主动极**：大语言模型 LLM（创造性、概率性生成）
- **受动极**：空模型 + 元事实库（确定性、规则性约束）

两者在结构化辩论层中形成张力，由裁决 AI 在「有依据随机」机制下做出选择，最终经人工审计/外部锚点行使最高否决权。

### 与 NMP 的关系

当前开源的 NMP 代码实现了 TDA 第一层「空模型校验层」的核心功能：

- ✅ 事实库检索与约束注入
- ✅ LLM 输出后置冲突校验
- ✅ 判例库自动积累
- ✅ <5ms 延迟、<100MB 存储、100% 可解释

第二层「结构化辩论层」和第三层「元受动极层」已具备完整理论设计。

TDA 架构的完整实现需要跨行业、跨领域的长期协作：

- **第一层（NMP）**：通用校验框架，已完成参考实现
- **第二层（辩论层）**：需要 NLP、形式逻辑、法学等多领域专家共建冲突图构建算法
- **第三层（裁决层）**：需要伦理学、监管科技、行业专家共建动态阈值模型

> *欢迎社区参与共建*。

## 核心差异

| | 传统 AI | NMP |
|--|---------|-----|
| 输入 | 问题 | 问题 + 事实库 |
| LLM 角色 | 直接回答 | 在事实边界内建议 |
| 安全兜底 | 无 | 校验层拥有最终否决权 |
| 可解释性 | 黑箱 | 100%（每次拦截输出事实 ID / 判例 ID） |

## 快速开始

```bash
# 1. 克隆仓库
git clone https://github.com/dragonwang725-design/null-model-Paradigm-v4.0.0.git
cd nmp

# 2. 安装依赖
pip install -r requirements.txt

# 3. 准备事实库
mkdir data
echo "塑料杯不耐高温，盛装热水会变形。" > data/vault_robot.json

# 4. 启动
python nmp.py
# 打开浏览器访问 http://127.0.0.1:7860
```

## 架构

```
用户问题
   ↓
【你有什么】事实库检索 → 提取相关客观事实
   ↓
LLM 在事实边界内生成策略
   ↓
【你有没有胡说】校验层核对事实冲突
   ↓
通过 / 拦截 / 重构
```

## 关键指标

- 校验延迟：<5ms
- 存储占用：<100MB（万级事实 + 十万级判例）
- 模型无关：可插拔任何 LLM
- 私域适配：新场景录入事实，10 分钟运行，无需重训练

## 场景示例

| 场景 | 传统 LLM | NMP |
|------|---------|-----|
| 机器人倒水 | 抓塑料杯接热水 → 变形/烫伤 | 事实库拦截"塑料杯 + 高温" → 重构为玻璃杯隔热接水 |
| 自动驾驶 | 白色塑料袋 → 触发 AEB 急刹 → 追尾 | 事实库"柔性/无质量" → 不触发 AEB |
| 雨天弯道 | 建议 55km/h | 事实库约束"摩擦<0.7 时车速≤48" → 拦截，不接受推理推翻 |
