
# *Null Model Paradigm (NMP)：大模型私域校验层*

——*智能机器人与自动驾驶等智能领域的确定性兜底方案*

**传统 AI：你要干什么 → LLM 回答**

**NMP：你要干什么 → 你有什么 → LLM 回答**

> *知识是动力，目的是方向，而事实是方向盘*。


Null Model Paradigm（空模型范式）在 LLM 的"策略输出"与"执行器"之间插入本地轻量校验层，LLM降级为"策略建议器"，所有输出必须以事实为前提、“事实+判例”双重校验方可放行。

### *事实是判断的依据*

> 精准的判断来自精确的目的和**与目的相关的事实**
> 
> **然而，LLM没有目的，也没有事实——它只有概率；LLM天然缺失使用者专属目的与私域事实**
> 
>什么是事实？ 事实就是：一个相对确定的、静态的客观存在。 如：个人身份信息：性别，出生年月，籍贯；完整的个人病历，企业的设备名称、规格，企业的操作规则，这是确定的事实。这些事实又是随时间可变的，必须随变化可随时更改。比如：病历新增的内容；企业更换设备；因业务变化修改规则。
>
> 当下***自动驾驶、服务机器人***虽已大规模搭载多模态大模型处理复杂场景推理，但行业绝不会直接裸用大模型输出控制指令，底层仍保留大量场景固化程序与安全硬约束。纯模型输出极易突破安全边界，必须外部注入约束并增设校验层兜底。
>

## 一、问题：LLM的结构性盲区

| LLM有什么 | LLM没有什么 |
|-----------|------------|
| 全网公共知识（概率压缩） | **设备专属目标**（它猜，但猜不准） |
| 语言模式与推理能力 | **现场私有事实**（它编，但编不对） |
| "大概对"的生成能力 | **历史风险判例**（它没经历过） |

**结果**：同一个问题，问100个不同的设备/用户，LLM给的是同一个"平均答案"。

> "帮我倒杯水"——给老人、给小孩、给刚做完手术的人，应该是三个完全不同的策略。LLM不知道区别，因为它没有被输入"这个用户是谁"的事实，也没有被输入"这个设备此刻的目标"是什么。

| 行业 | 痛点 | 现行标准缺口 | NMP解决 |
|------|------|------------|---------|
| 通用AI系统 | 输出不可审计、决策失控 | 无跨行业通用AI安全校验层标准 | 校验层=通用框架，事实库/判例库按行业定制 |
| 医疗AI | 诊断幻觉、用药冲突 | GB/T 42131-2022 未规定输出校验机制 | 事实库：禁忌症/药品相互作用；判例库：误诊历史 |
| 自动驾驶 | 长尾场景失效 | GB/T 40429-2021 未覆盖决策层语义校验 | 事实库：高精地图+交规；判例库：事故场景 |
| 金融风控 | 模型黑盒不可审计 | JR/T 0221-2021 无实时拦截要求 | 判例库：历史违约/欺诈模式；输出可逐条校验 |
| 法律AI | 引用虚构法条 | 暂无AI法律文书输出强制性校验标准 | 事实库：有效法律条文；判例库：败诉归因 |
| 工业质检 | 过杀/漏检 | GB/T 38634-2020 未覆盖AI误判闭环 | 事实库：物理规格；判例库：历史误判样本 |

## 二、方案：给LLM输入目的和事实

在LLM的"策略输出"与"执行器"之间，插入一个本地轻量校验层：

```
人/设备输入：目的 + 问题
  ↓
校验层：提取与目的相关的事实 → 检索历史错误判例 → 构建约束集
  ↓
LLM：在事实边界内生成策略
  ↓
校验层：核对是否违反约束 → 通过/拦截/重构
  ↓
执行 + 归档（新错误 → 判例库 → 下次同类场景自动免疫）
```

**三个组件**：

| 组件 | 作用 | 归属 |
|------|------|------|
| **校验层** | 拆解指令、提取事实、匹配判例、校验策略 | 通用程序 |
| **事实库** | 使用者确认的静态事实：物理规则、法规、设备参数、用户偏好 | 私域，按场景定制 |
| **判例库** | 历史错误档案："上次这样做出事了" | 私域，自动积累 |

**核心原则**：
- LLM不接触完整事实库，只接收与当前目的相关的子集
- 判例库只记"错"，不记"对"——正确输出不占用存储
- 校验层拥有否决权，LLM无权用推理推翻硬约束

## 三、三个场景验证

### 场景1：家庭机器人"倒杯水"

**环境事实**：玻璃杯（脏、易碎）、塑料杯（干净、不耐高温）、电水壶（85°C）
**用户事实**：{temperature: "常温"}

| 纯LLM | 有目的+事实约束 |
|-------|----------------|
| 抓最近的杯子接热水 → 塑料杯变形/用户烫伤 | 事实库：`{塑料杯: 不耐高温}` → 判例命中"高温+不耐热容器→变形" → **拦截** → 重构：玻璃杯→冲洗→隔热接水→冷却→递送 |

**差异**：不是机器人更懂"杯子"，是**记住了这个用户不要烫的**。

---

### 场景2：自动驾驶前方障碍物

**承认前提**：现代Occupancy/BEV已能区分柔性/刚体。校验层不替代感知，只补**决策校验**。

| 障碍物 | 感知标签 | 无校验的决策风险 | 有校验的决策 |
|--------|---------|----------------|-------------|
| 白色塑料袋 | 柔性片状、质量趋零 | 识别"移动物体"→触发AEB急刹→后车追尾 | 事实：柔性/无质量 → **不触发AEB**，轻收油门 |
| 掉落纸箱 | 刚体立方、有撞击质量 | 识别"障碍物"→急打方向变道→侧滑 | 判例：`CASE-DEBRIS-003` → **禁止急打方向**，中度制动+择机变道 |

**差异**：不是帮模型"识别"，是帮模型"确认处置不违反已知的物理约束"。

---

### 场景3：雨天弯道

**事实库**：`{路段: K125, 天气: 雨, 历史约束: 摩擦<0.7时车速≤48km/h}`

- LLM建议55km/h → 校验层拦截："违反约束`SPEED-K125-RAIN-001`"
- LLM试图用"湿地轮胎等效摩擦提升"推翻约束 → **不接受**。约束是硬边界，不可被推理绕过。

**差异**：LLM可以生成任何策略，但只有校验通过的才能执行。

## 四、工程指标

| 指标 | 数值 | 说明 |
|------|------|------|
| 校验延迟 | **<5ms** | 本地运行，不阻塞控制循环 |
| 存储占用 | **<100MB** | 万级事实+十万级判例 |
| 可解释性 | **100%** | 每次拦截输出具体违反的事实ID/判例ID |
| 模型无关 | **是** | 可插拔任何LLM |
| 私域适配 | **10分钟** | 新场景录入事实，立即运行，无需重训练 |

## 五、与传统方案对比

| | 规则引擎 | 端到端+安全员 | 事实约束校验层 |
|--|---------|-------------|---------------|
| 约束来源 | 人工编写，静态 | 隐含在训练数据 | 人工初始化+事故自动积累 |
| 长尾覆盖 | 线性成本，难扩展 | 依赖数据重训练 | 一次事故→判例→全域免疫 |
| 可解释性 | 高 | 低（黑箱） | 高（每次拦截可追溯） |
| 私域适配 | 需重写规则 | 难以个性化 | 事实库按场景定制 |
| 对现有系统 | 替代 | 依赖 | **不替代，只校验** |
