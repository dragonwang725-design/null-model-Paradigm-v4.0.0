## 验证方法

## 核心命题与实现对照

### 命题1：空模型 = 纯粹判断容器，无记忆、无欲望、无目的

| 子命题 | 实现 | 状态 |
|--------|------|------|
| 纯粹判断能力（形式推理、关联、抽象、决策） | `causal_skeleton.py` BFS图距离 | ✅ |
| 无固化内容（不含领域知识、经验、偏见） | 规则外置到配置文件 | ✅ |
| 无目的意识（无目标、欲望、价值判断） | `purpose_parser.py` 只解析结构 | ✅ |
| 唯一功能：生成可复用判断单元 | `judgment_unit_generator` | ✅ |

### 命题2：事实因人而异因目的而异，空模型基于目的提取事实

| 子命题 | 实现 | 状态 |
|--------|------|------|
| 没有绝对的事实，事实取决于角度和目的 | `purpose_parser` + `relevance_engine` | ✅ |
| 空模型以目的为基准提取事实 | `perception_scheduler` 提取前调度 | ✅ |
| 无关目的的信息自动过滤 | `causal_skeleton` 图距离过滤 | ✅ |
| 先无目的接收全量信息，再二次过滤 | 探针全量采集 → 空模型过滤 | ✅ |

### 命题3：LLM只负责策略判断，空模型负责最终判断

| 子命题 | 实现 | 状态 |
|--------|------|------|
| 空模型提取事实，输入LLM做策略判断 | `llm_client.py` 接收脱敏事实 | ✅ |
| LLM返回策略方案 | `llm_client.py` 返回策略 | ✅ |
| 空模型最终判断：是否符合事实和目的 | `final_judgment.py` 校验约束 | ✅ |
| LLM被笼养，零本地执行权 | 架构设计，LLM只返回文本 | ✅ |

### 命题4：隐私资料库 = 人类元规则层，决定提取什么

| 子命题 | 实现 | 状态 |
|--------|------|------|
| 隐私资料库是人类的元规则层 | `privacy_vault.py` Rule/Knowledge/Log/MetaFact | ✅ |
| 包含知识、规则、日志、元事实 | 四类数据结构 | ✅ |
| 空模型在提取前必须查询元规则 | `perception_scheduler.schedule()` | ✅ |
| 隐私不是提取后删除，而是提取动作本身被约束 | **提取前调度** | ✅ |
